HTML

  • Content is added and organized using elements, which consist of a starting and ending tag and everything in between. A tag is enclosed in angle brackets (<>), and an ending tag begins with a slash (</>)
  • The !DOCTYPE declaration tells the browser to expect an HTML file
  • The rest of the page elements are contained within the html element with a lang="en" attribute
  • The head element contains information about the webpage.
    • charset
    • http-equiv for compatibility
    • viewport, which enables responsiveness
    • title, which displays in the browser tab
  • The body element represents the visible content shown to the user.
    • Content can be divided with section, div, and span elements. Semantic elements, such as section, are preferred.
    • Headings include h1 through h6. Each page should have exactly one h1.
    • Lists include ordered (ol) and unordered (ul) lists, each consisting of list items (li).

Git

1. Check the current branch. git status
2. Add a branch and switch to that branch. git checkout -b [type (e.g., feature)]/[name]
Alt Switch to a branch. git checkout [type]/[name]
3. Check the current branch again. git status
4. Stage (add) files in the current repository to the current branch. git add -A
OR git add [filename]
5. Commit staged files to the next push. git commit -m "[commit message]"
6. Ensure the local branch is synchronized with the base branch in GitHub. git pull origin main
7. Push changes to the remote branch. git push origin [type]/[name]

Next, navigate to the GitHub repo in the browser and select the desired branch.

  1. Click Compare & pull request.
  2. Add a descriptive title.
  3. In the body, add instructions for team members who will review the PR:
    1. What changes were made
    2. How the feature is supposed to function
    3. How to start the application so the feature can be tested
  4. Designate the base and compare branches.
  5. Click Create pull request.
